# 代码风格规范

本项目后端代码严格遵循PEP8编码规范，旨在保证代码的可读性、一致性和可维护性。以下是主要遵循的规范要点：

## 1. 缩进
- 使用4个空格进行缩进，不使用制表符（Tab）
- 续行缩进应额外增加4个空格

## 2. 命名规范
- 变量、函数、方法、模块：使用小写字母，单词之间用下划线连接（蛇形命名法），如`user_model`、`create_app`
- 类名：使用首字母大写的驼峰命名法，如`UserModel`、`Config`
- 常量：使用全大写字母，单词之间用下划线连接，如`SECRET_KEY`

## 3. 代码行长度
- 每行代码最大长度不超过79个字符
- 长表达式需换行，换行后缩进对齐

## 4. 导入规范
- 导入顺序：标准库 → 第三方库 → 本地模块，不同类别之间用空行分隔
- 避免使用`from module import *`的导入方式
- 导入语句单独成行，如：
  ```python
  import os
  from datetime import timedelta
  from flask import Flask
  
## 5. 注释
- 模块、类、函数需使用文档字符串（三引号）进行说明，描述其功能、参数和返回值
- 单行注释需在代码后至少保留两个空格，再添加注释内容
- 注释应简洁明了，避免冗余或无意义的注释

## 6. 空行
- 函数和类之间空两行
- 类内部方法之间空一行
- 逻辑相关的代码块之间可适当使用空行分隔
- 模块顶部导入后，函数 / 类定义前可使用空行分隔

## 7. 其他规范
- 运算符两侧各保留一个空格，如a = b + c
- 逗号后保留一个空格，如(1, 2, 3)
- 避免在行尾添加空格
- 使用显式的相对导入或绝对导入，避免隐式导入
